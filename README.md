# secure-temporary-payment-system
A simulated secure payment gateway for online games designed to reduce exposure of sensitive information through temporary payment sessions and expiring reference codes.

## Overview
This project is a simulated secure payment gateway for online games. It is designed to provide a temporary payment window that acts as an extra protection layer for sensitive information during digital game purchases.

## Features
- Player registration and login
- Game purchase flow
- Temporary payment session
- Expiring reference code
- Confirm/cancel payment flow
- Transaction status tracking
- Admin dashboard
- Audit logging

## Technologies
- Java EE 8
- Servlet/JSP
- MySQL
- GlassFish
- NetBeans

## Security Focus
- Minimal payment-related data exposure
- Expiring payment sessions
- Role-based access control
- Password hashing
- Server-side validation
- Safe transaction logging

## Project Goal
The goal of this project is to demonstrate how a temporary secure payment workflow can reduce exposure of sensitive information in online game purchase processes.
